# Summary

PyStamps is an open-source Python package for measuring ground deformation from time series of Synthetic Aperture Radar (SAR) acquisitions using the Persistent Scatterer Interferometry (PSI) technique. It implements the Stanford Method for Persistent Scatterers (StaMPS) workflow [@hooper2012], and is designed to operate on interferometric stacks generated by common preprocessing tools such as ESA’s SNAP toolbox [@foumelis2018].

The package identifies persistent scatterers—radar targets exhibiting stable phase behaviour over time—and derives high-precision line-of-sight (LOS) deformation time series suitable for monitoring subsidence, infrastructure deformation, and tectonic motion.

Unlike the original MATLAB-based StaMPS implementation, PyStamps removes proprietary software dependencies and provides a modular Python-based implementation of the PSI processing chain. It preserves the validated StaMPS methodology while integrating modern scientific computing tools, including NumPy- and SciPy-based vectorised computation [@harris2020array; @virtanen2020scipy], patch-based parallel processing, and HDF5-based intermediate data storage for large-scale datasets.

# Statement of need

Persistent Scatterer Interferometry (PSI) enables millimetre-scale deformation monitoring from repeated SAR acquisitions and is widely used in geohazard assessment, infrastructure monitoring, and geotechnical applications [@crosetto2016persistent; @ramirez2020long]. The increasing availability of Sentinel-1 data has created a need for scalable, reproducible, and open-source PSI processing workflows.

The Stanford Method for Persistent Scatterers (StaMPS) is a widely adopted PSI framework [@hooper2012], but its MATLAB dependency introduces licensing constraints and limits integration into modern cloud-native and automated geospatial pipelines. While several InSAR toolboxes exist, there is currently no complete open-source Python-native implementation of the StaMPS PSI chain suitable for operational-scale processing.

PyStamps addresses this gap by providing a Python implementation of the StaMPS PSI workflow compatible with interferometric products generated by SNAP and similar preprocessing toolchains. The system is designed for scalable Sentinel-1 time-series analysis and supports both research and operational deformation monitoring workflows.

# Implementation and features

PyStamps implements the full StaMPS PSI workflow, including:

- persistent scatterer candidate selection  
- phase estimation and filtering (weeding)  
- patch-based processing and merging  
- three-dimensional phase unwrapping  
- atmospheric phase screen (APS) estimation and correction  
- spatially correlated look angle (SCLA) correction  
- inversion for line-of-sight (LOS) displacement time series  

The architecture is modular and built on Python scientific computing libraries. Intermediate products are stored in HDF5 format to enable efficient handling of large interferometric stacks. Computational performance is improved through vectorised operations and parallel processing at the patch level, enabling scalability across multi-core and distributed/cloud environments.

By preserving the validated StaMPS methodological framework while removing MATLAB dependencies, PyStamps provides a reproducible and extensible platform for PSI-based deformation analysis.
